前言

本文章主要是将日常SpringBoot项目中用到的注解记录下来。

SpringBoot

注意：注解中只有value的时候 value是可以省略的 例如 @JsonProperty(value=”xxx”) 中的value可以省略！

启动类

  • @SpringBootApplication
    • @ComponentScan：默认是扫描启动类所在包及其子包的Component组件
    • @Import(XxxxSelector.class)
    • @Documented
    • @Inherited: 暂时不是很能理解这个注解的作用
    • @SpringBootConfiguration
    • EnableAutoConfuguration
  • @MapperScan(“com.briup.dao”)
  • @EnableSwagger2Doc：原生Swagger
  • @SpringBootTest：测试类里面
  • @AutoConfiguration：JacksonObjectMapper
  • @Conditional
    • @ConditionalOnClass：判断环境中有对应字节码文件，才注册bean到IOC容器
    • @ConditionalOnMissingBean：判断环境中没有对应的bean对象（类型或名称），才注册bean到IOC容器
    • @ConditionalOnProperty：判断配置文件application.yml中有对应属性和值，才注册bean到IOC容器、
  • @EnableCaching；开启缓存注解功能
    • @Cacheable：在方法执行前先查看缓存中是否有数据，如果有数据，则直接返回缓存数据；否则调用方法并将方法返回值放到缓存中
    • @CachePut：将方法的返回值放到缓存中 （只放不取）
    • @CacheEvict：将一条或多条数据从缓存中删除

Config 配置类

  • @Configuration
  • @EnableSwagger2WebMvc: Knife4jConfiguration
  • @ConfigurationProperties(prefix = “oss”)：可以批量的将外部的属性配置注入到bean对象的属 性中。
  • @ServletComponentScan: 开启对Servlet组件的支持
  • @EnableSwagger2：Swagger2Config配置类
  • @Bean：将方法返回结果注册到IOC容器中

Pojo 实体类

  • @Data
    • @Getting
    • @Setting
    • @ToString(callSuper=true)：可以调用父类的toString方法
    • @EqualsAndHashCode
  • @NoArgsConstructor
  • @AllArgsConstructor
  • @Builder
  • @Value(“${oss.accessKey}”)：将配置文件.yml与实体类一 一对应。注解只能一个一个的进行外部属性的注入。
  • @Accessors(chain = true)开启链式开发

具体使用看看这篇优质博客：Lombok插件@Accessors(chain = true)开启链式开发

  • @JsonProperty(“id”)：保证序列化和反序列化的时候前后端属性名字能够对应上
  • @JsonFormat(pattern = “yyyy-MM-dd HH:mm:ss” ,timezone = “GMT+8”)
  • ApiModel：类上添加描述信息
  • ApiModelProperty：方法上添加描述信息
  • @RequiredArgsConstructor ：Lombok注解：构造器注入：@RequiredArgsConstructor_@required
  • @TableFile(value, exist, select)
  • @TableName
  • @TableId(type = IdType.xxx)

  • @TableLogic(value=”0”, delval=”1”)
  • @JsonSerialize(using = ToStringSerializer.class) ：后端传给前端Long->String,前端传后端String->Long
  • @ExcelIgnore
  • @ExcelProperty(value = “栏目删除状态”, converter = DeletedConverter.class)
  • @JsonIgnore
    • 作用：在json序列化时将java bean中的一些属性忽略掉，序列化和反序列化都受影响。
    • 使用方法：一般标记在属性或者方法上，返回的json数据即不包含该属性。

Filter 过滤类

  • @WebFilter(urlPatterns = “/*”)：/*表示拦截所有请求

Aspect 切面类

  • @Aspect
    • @Around(@annotation, execution) ：方法返回值必须是Object
    • @Before
    • @After
    • @AfterReturning
    • @AfterThrowing
    • @Pointcut
  • @Order(1) //切面类的执行顺序（前置通知：数字越小先执行; 后置通知：数字 越小越后执行, 两两对应）

Other类

  • @RequestMapping
    • @GetMapping
    • @PostMapping
    • @PutMapping
    • @DeleteMapper

Put和Post的区别：https://blog.51cto.com/u_15964010/6246766

  • @RestController
    • @Controller
    • @ResponseBody
  • @RequestParam
    • required：是否为必传参数，默认为true，必须传递该参数
    • defaultValue：参数默认值

http://localhost:8081/simpleParam?name=zs&age=21

绑定请求参数与处理器方法形参间的关系 即 ?后面的参数数据

结论：请求参数名与形参名不同时，使用@RequestParam注解关联请求参 数名与形参名

  • @RequestPart: 主要用于文件 MultipartFile类型参数

  • @DateTimeFormat

    • 前端必须按照@DateTimeFormat注解的pattern属性中指定的日期格式进行 参数传递。
    • 后端Controller方法中，需要使用**Date或LocalDateTime(建议)**类型，来封装传递的 参数。

  • @RequestBody
  • @PathVariable

路径参数：http://localhost:8081/deleteById/1

  • @RestControllerAdvice：Advice增强类 其实就是Aop类
    • @ControllerAdvice
    • @ResponseBody
  • @ExceptionHandler

这篇博客写的很好：RestControllerAdvice注解与全局异常处理

  • @Component
    • @Controller
    • @Service
    • @Repository：Spring原生的 ==@Mapper Mybatis的
  • @Api(tags = “Swagger测试”)
  • @ApiOperation(notes=”swagger测试”, value=””)
  • @ApiImplicitParams()
    • @ApiImplicitParam(name, value, required, paramType, dataType)
  • @Autowired
  • @Resource
  • @Qualifier：指定注入的对象

  • @Param(“keyword”) String keyword
  • @Slf4j
  • @Test：Junit
  • @Transactional(rollbackFor: 异常回滚的属性,propagation：事务传播行为)
  • @SuppressWarnings(“unchecked”)：取消检查
  • @Scope：取值“singleton”or “prototype”：当取值苇prototype是会默认加上@Lazy
  • @Lazy：延迟加载, 可以延迟bean对象的初始化，延迟到第一次使用时创建
  • @ServerEndpoint(“/ws/{sid}”)：Websocket
    • @OnOpen：连接成功调用的方法
    • @OnMessage：收到客户端消息后的方法
    • @OnClose：连接关闭调用的方法

SpringCloud

  • @EnableEurekaServer

  • @EnableFeignClients(“com.briup.cloud.web.rest”)

  • @EnableHystrix

  • @EnableDiscoveryClient

  • @EnableZuulProxy

  • @LoadBalanced

  • @FeignClient(name=”producer”)

  • @SentinelResource(value=”xxxx”, blockHandler=”xxxx”)
